# 農産物 在庫管理システム / Produce Inventory Management System

> 農産物加工事業の現場で使うことを想定した、**オンプレミス完結型（オフラインファースト）の在庫管理 Web アプリ**。
> 入荷ロット管理・FIFO 出庫・選別/歩留・資材レシピ・棚卸・倉庫レイアウト（3D）・帳票出力までを一気通貫で扱う。

このリポジトリは**ポートフォリオ用の公開版**です。実在の社内データ・取引先・人物・設備名は一切含まれず、
すべて `tools/seed_demo.py` が生成する**架空の合成データ**で動作します（[サンプルデータ](#サンプルデータの投入) 参照）。

## 🚀 ライブデモ

**▶ https://produce-inventory-demo.onrender.com**  ←（デプロイ後に実URLへ差し替え）

- スマホ・iPad・PC のブラウザからそのまま触れます。
- 認証はデバイス登録方式ですが、**デモ環境では登録した端末を自動承認**します（任意の表示名で「登録」すれば即利用可）。
- 表示されるデータはすべて合成サンプルです。自由に入出庫・選別などを試せます。
- 無料ホスティングのため、しばらくアクセスが無いとスリープし、初回表示に数十秒かかることがあります。

## なぜこの構成にしたのか（設計判断）

このシステムの中心にあるのは **「間違えようがない、だから安心して速く動ける」** という思想です。
在庫は「数字が1つズレると現場が止まる」領域なので、正しさをアプリ任せにせず**データベース層で担保**しました。

| 判断 | 採用 | 理由 |
|------|------|------|
| **DB は PostgreSQL** | SQLite ではなく PostgreSQL | 在庫の整合性は FK・CHECK 制約・**トリガー**・ビューで DB 自身に守らせる。たとえば「入荷日より前の出庫」「在庫量を超える出庫」は `check_stock_before_outbound` トリガーが物理的に拒否する。アプリのバグでデータが壊れない設計。 |
| **オフラインファースト / オンプレ完結** | Docker 不要・クラウド非依存 | 加工現場はネットワークが不安定でも止められない。**Windows PC 1台で完結**して動くことを最優先にした（外部公開は Tailscale 等で別途）。 |
| **バックエンドは FastAPI + psycopg3 (async)** | 同期 ORM ではなく非同期素朴 SQL | 在庫計算は複雑な集計クエリが多い。ORM の抽象に隠さず**SQL を読める形で書く**ことで、FIFO や歩留の計算ロジックを正確に制御。Pydantic v2 で入出力を型で固める。 |
| **PDF は「印刷最適化ページ + ブラウザ印刷」** | サーバーサイド PDF エンジンを使わない | 現場が長年使ってきた Excel 帳票の見た目を**ピクセル単位で再現**する必要があった。React の印刷専用ページ（CSS `@page` をコンテンツに合わせて自動調整）＋ `window.print()` で、重い依存を増やさず “紙に出る形” を完全制御。 |
| **Excel との往復を残す** | openpyxl で `.xlsx`/`.xlsm` を生成・取込 | 現場の運用は Excel 台帳が前提。**いきなり全置換せず**、既存の Excel 台帳をシステム在庫で同期し直して返す機能（大蒜仕入管理台帳の同期）で、移行の段差をなくした。VBA で書かれていた計算ロジックを Python に移植している。 |

> 設計の発展方針（SQLite ローカルレプリカによる多拠点オフライン同期、サーバーサイド PDF 生成など）は
> [docs/ROADMAP.md](docs/ROADMAP.md) に分離して記述しています（**現行の稼働版は上記の PostgreSQL 構成**）。

## 技術スタック

**バックエンド**
- Python 3.12 / **FastAPI**（async）
- **PostgreSQL 16** / **psycopg 3**（async, コネクションプール）
- Pydantic v2 / openpyxl（Excel 帳票生成・取込）
- pytest / pytest-asyncio（ドメインロジックの単体テスト）

**フロントエンド**
- **React 19** / **TypeScript** / **Vite**
- React Router 7（SPA）
- **three.js + react-three-fiber + drei**（倉庫レイアウトの 3D 表示）
- react-spring（アニメーション）/ lucide-react（アイコン）

**アーキテクチャ（3層）**

```
┌─────────────────────────────────────────────┐
│  Presentation : React SPA (+3D, 印刷ページ)    │  ← ブラウザ / iPad
├─────────────────────────────────────────────┤
│  Application  : FastAPI (async) + ドメイン層    │  ← FIFO割当 / 選別歩留 / レシピ推定 / 振替
├─────────────────────────────────────────────┤
│  Data         : PostgreSQL (制約/トリガー/ビュー)│  ← 整合性を DB 自身が保証
└─────────────────────────────────────────────┘
```

## 主な機能

- **入荷ロット管理**：ロット単位の入荷・単価確定・前払/後払・諸経費（仲介料/運賃）の記録
- **FIFO 出庫**：先入れ先出しでの引き当て（`fifo_eligible_lots` ビュー＋割当サービス）
- **選別・歩留**：原料ロットを規格別に選別し、歩留・廃棄を追跡（選別 → 半製品 → 出荷の連鎖）
- **資材レシピ**：商品1点あたりの資材消費量を定義し、出荷から資材理論在庫を逆算。代替資材・推定レシピにも対応
- **棚卸**：月次棚卸（理論値 vs 実数）、置き場単位の棚卸エントリ
- **倉庫レイアウト（2D/3D）**：パレット・コンテナを倉庫平面に配置し、3D で可視化
- **帳票出力**：仕入管理台帳・ダッシュボードの印刷用 PDF、各種集計の Excel エクスポート
- **権限分離 / デバイス承認認証**：viewer / operator / admin / recipe_editor のロール、デバイストークン方式
- **監査ログ・訂正履歴**：過去データの修正を追跡

## セットアップ

### 前提
- Python 3.12 / PostgreSQL 16 / Node.js 18+（フロントエンド開発用）
- Windows を主対象（`*.ps1` スクリプト同梱）。他 OS でも手順は同等。

### 1. データベースとスキーマ

```powershell
# DB 作成
psql -U postgres -c "CREATE DATABASE inventory_db;"
# スキーマ適用（47テーブル / 5ビュー / 7関数 / トリガー）
psql -U postgres -d inventory_db -f db\schema.sql
```

### 2. 環境変数

```powershell
Copy-Item .env.example .env
# .env を開き、DATABASE_URL の password をご自身の PostgreSQL パスワードに変更
```

### 3. バックエンド

```powershell
.\setup.ps1        # 仮想環境作成 + requirements インストール
```

### 4. サンプルデータの投入

```powershell
.\.venv\Scripts\python.exe tools\seed_demo.py
```

合成データ（生姜・大蒜の在庫、架空の取引先・資材・出荷など）が投入されます。
再実行しても重複しません（冪等）。クリアしたい場合は `tools\clear_samples.py`。

### 5. 起動

```powershell
# バックエンド（http://localhost:8000, Swagger UI: /docs）
.\start.ps1

# フロントエンド（別ターミナル, http://localhost:5173）
cd frontend
npm install
npm run dev
```

### 6. 初回ログイン

認証はデバイストークン方式です。**管理者が1人も登録されていない初回**は、
最初にデバイス登録した利用者が**自動的に管理者として承認**されます。
ブラウザで開いて任意の表示名で登録すれば、そのまま管理者として全機能を利用できます。

## プロジェクト構成

```
.
├── api/                  # FastAPI バックエンド
│   ├── main.py           #   エンドポイント / SPA 配信 / 帳票出力
│   ├── routers/          #   機能別ルーター（入出庫・資材・選別・倉庫・資産…）
│   ├── services/         #   ドメインロジック（FIFO割当・歩留・振替・レポート）
│   ├── auth.py           #   権限分離（viewer/operator/admin/recipe_editor）
│   └── dependencies.py   #   async コネクションプール
├── db/
│   └── schema.sql        # 単一スキーマ（テーブル/ビュー/関数/トリガー）
├── frontend/             # React + Vite + TypeScript SPA
│   └── src/pages/        #   各画面（3D 倉庫・カレンダー・帳票印刷…）
├── tools/
│   ├── seed_demo.py      # 合成サンプルデータ投入（オーケストレータ）
│   ├── seed_*.py         #   作物・資材・出荷の各シード
│   └── clear_samples.py  # サンプルデータのクリア
├── tests/                # pytest（ドメインロジックの単体テスト）
└── docs/ROADMAP.md       # 設計の発展方針
```

.\.venv\Scripts\python.exe -m pytest
